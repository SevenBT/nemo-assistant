# 需求文档：模型设置与自定义提示词

## 背景

当前项目的 System Prompt 硬编码在 `main_window.py` 中，模型参数配置项较少。用户无法自定义 AI 行为风格、调整高级参数、或针对不同会话使用不同的提示词。

参考产品：ChatGPT（自定义指令）、Claude Desktop（System Prompt）、Cherry Studio（多模型/多预设）、NextChat（角色面具）、LobeChat（助手市场+高级参数）、Open WebUI（模型文件/高级参数面板）。

---

## 功能模块

### 一、全局 System Prompt 设置 ⭐

**目标**：用户可在设置中自定义全局 System Prompt，作为所有新会话的默认提示词。

**现状**：`SYSTEM_PROMPT` 硬编码在 `main_window.py:55`，`Session` 模型已有 `system_prompt` 字段但未使用。

| 项 | 说明 |
|----|------|
| 入口 | 设置 → 新增「模型」Tab |
| 默认值 | 当前硬编码的 SYSTEM_PROMPT 内容 |
| 编辑器 | QPlainTextEdit，支持多行编辑，高度 ≥ 200px |
| 存储 | `config/app_config.json` → `api.system_prompt` |
| 重置 | 提供「恢复默认」按钮 |
| 生效 | 仅对新会话生效，已有会话不受影响 |
| 工具说明保护 | 内置工具使用说明自动追加到用户自定义 Prompt 后 |

**参考**：
- ChatGPT「自定义指令」— 全局配置，新会话自动带入
- Cherry Studio「默认助手设置」— 全局 System Prompt + 每个助手可覆盖

**用户价值**：直接改变 AI 行为风格（专业/幽默/简洁），体感明显。

---

### 二、会话级 System Prompt 覆盖

**目标**：每个会话可独立设置 System Prompt，覆盖全局默认值。

| 项 | 说明 |
|----|------|
| 入口 | 聊天界面顶部或右键菜单 → 「会话设置」 |
| 优先级 | 会话级 > 全局级 |
| 存储 | `Session.system_prompt` 字段（已存在） |
| 显示 | 会话列表标记有自定义 Prompt 的会话（如小图标） |

**参考**：
- NextChat「面具」— 每个面具绑定独立 System Prompt
- LobeChat — 会话级参数覆盖

---

### 三、预设角色（Prompt 模板）⭐

**目标**：预置一组常用角色模板，用户可一键应用，也可自定义创建。

#### 内置模板示例

| 角色名 | 简述 | 核心 Prompt 方向 |
|--------|------|------------------|
| 默认助手 | 通用 AI 助手 | 当前默认 Prompt |
| 翻译官 🌐 | 中英互译 | 检测输入语言，翻译为另一种语言 |
| 代码助手 💻 | 编程辅助 | 专注代码生成和技术问题 |
| 写作助手 ✏️ | 文案创作 | 擅长各类文体写作和修改 |
| 摘要大师 📋 | 内容总结 | 精炼概括长文内容 |

#### 自定义模板

| 项 | 说明 |
|----|------|
| 管理入口 | 设置 → 模型 Tab → 「管理预设角色」 |
| 字段 | 名称、图标（可选 emoji）、System Prompt、默认参数覆盖 |
| 操作 | 新建 / 编辑 / 删除 / 导入 / 导出 |
| 存储 | `config/presets.json` |
| 新建会话 | 新建会话时可选择预设角色 |

**参考**：
- NextChat「面具商店」— 模板列表 + 一键使用
- Cherry Studio「助手」— 角色模板 + 参数绑定
- LobeChat「助手市场」— 社区共享 + 本地创建

**用户价值**：比调参数直观 100 倍，一键切换"翻译官""写作助手"，降低使用门槛。

---

### 四、多模型配置

**目标**：支持配置多个模型端点（API 地址 + Key + 模型名），会话中可快速切换。

| 项 | 说明 |
|----|------|
| 管理入口 | 设置 → API Tab → 「管理模型」 |
| 每条配置 | 名称、API 地址、API Key、模型名、默认参数 |
| 切换方式 | 标题栏下拉或会话设置中选择 |
| 会话绑定 | 每个会话记录使用的模型配置 ID |
| 存储 | `config/models.json` |

**参考**：
- Cherry Studio — 多服务商配置（OpenAI/Claude/Gemini/本地）
- NextChat — 多模型快速切换
- LobeChat — 模型管理器 + 会话绑定

## 优先级与分期

| 阶段 | 功能 | 优先级 | 复杂度 | 用户价值 |
|------|------|--------|--------|----------|
| P0 | 全局 System Prompt 设置 | 高 | 低 | ⭐⭐⭐⭐⭐ 直接改变 AI 行为 |
| P1 | 预设角色（Prompt 模板） | 高 | 中 | ⭐⭐⭐⭐⭐ 降低使用门槛 |
| P1 | 会话级 System Prompt 覆盖 | 中 | 中 | ⭐⭐⭐ 灵活性提升 |
| P2 | 多模型配置与切换 | 低 | 高 | ⭐⭐ 高级用户需求 |
| ~~砍掉~~ | ~~模型高级参数~~ | ~~低~~ | ~~低~~ | ⭐ 普通用户不理解 |

**砍掉理由**：top_p / penalties 等参数对普通用户无意义，ChatGPT 面向数亿用户也不暴露。现有 Temperature + Max Tokens 已覆盖 95% 场景。

建议分两期实施：
- **第一期（P0）**：全局 Prompt 设置 — 改动集中在 `settings_dialog.py` 和 `config.py`，风险低，用户价值最高
- **第二期（P1）**：预设角色 + 会话级覆盖 — 需改动会话管理和新建会话流程，用户体验提升明显
- **第三期（P2）**：多模型配置 — 架构变动较大，涉及 `AIClient` 重构，面向高级用户

## 数据结构变更

### config/app_config.json

```jsonc
{
  "api": {
    // 现有字段...
    "system_prompt": "你是一个智能AI助手..."   // 新增（P0）
  }
}
```

### config/presets.json（新文件）

```jsonc
[
  {
    "id": "default",
    "name": "默认助手",
    "icon": "🤖",
    "system_prompt": "你是一个智能AI助手...",
    "params": {}              // 空 = 使用全局参数
  },
  {
    "id": "translator",
    "name": "翻译官",
    "icon": "🌐",
    "system_prompt": "你是一个专业翻译...",
    "params": {
      "temperature": 0.3
    }
  }
]
```

### config/models.json（新文件，P2）

```jsonc
[
  {
    "id": "openai-gpt4o",
    "name": "GPT-4o",
    "base_url": "https://api.openai.com/v1",
    "api_key_ref": "openai_main",    // keyring 引用
    "model": "gpt-4o",
    "params": {}
  }
]
```

### Session 模型扩展

```python
@dataclass
class Session:
    # 现有字段...
    system_prompt: str = ""       # 已存在
    preset_id: str = ""           # 新增：预设角色 ID
    model_config_id: str = ""     # 新增（P2）：模型配置 ID
```

## 约束与注意事项

1. **向后兼容**：config 新字段都有默认值，升级后旧配置正常工作
2. **System Prompt 中内置工具说明不可丢失**：用户自定义 Prompt 时，定时任务/笔记等内置工具的使用说明需要自动追加（或作为不可编辑的后缀），避免工具调用失败
3. **API Key 安全**：继续使用 keyring 存储，不写入 JSON
4. **预设角色导入/导出**（P1）：使用 JSON 格式，方便社区共享
5. **普通用户优先**：功能设计面向大众用户，避免暴露过多技术参数
