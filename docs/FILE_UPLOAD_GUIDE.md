# 文件拖拽上传功能 - 快速使用指南

## 功能概述

现在你可以通过拖拽文件到聊天区域来上传文件，AI 会自动读取文件内容并进行分析。

## 使用步骤

### 1. 拖拽文件
- 从文件管理器中选择一个或多个文件
- 拖拽到聊天区域
- 松开鼠标

### 2. 查看附件
- 文件会显示为卡片形式
- 显示文件名、大小和图标
- 图片文件会显示缩略图

### 3. 发送消息
- 输入你的问题或指令
- 点击发送
- AI 会看到文件内容并进行回复

### 4. 打开文件
- 点击文件卡片可以用系统默认程序打开文件

## 支持的文件类型

| 类型 | 扩展名 | 处理方式 |
|------|--------|----------|
| 文本 | `.txt` | 直接读取内容 |
| Markdown | `.md` | 直接读取内容 |
| 图片 | `.png`, `.jpg`, `.jpeg` | OCR 识别文字 |

## 限制

- 单个文件最大 10MB
- 仅支持上述文件类型
- 图片 OCR 识别准确率取决于图片质量

## 示例场景

### 场景 1: 分析文本文件
1. 拖入 `report.txt`
2. 输入: "总结这份报告的要点"
3. AI 会读取文件内容并生成摘要

### 场景 2: 识别图片文字
1. 拖入 `screenshot.png`
2. 输入: "提取图片中的文字"
3. AI 会使用 OCR 识别图片中的文字

### 场景 3: 代码审查
1. 拖入 `main.py`
2. 输入: "检查这段代码有没有问题"
3. AI 会分析代码并给出建议

## 故障排查

### 文件无法上传
- 检查文件类型是否支持
- 检查文件大小是否超过 10MB
- 查看控制台日志了解详细错误

### OCR 识别失败
- 确保图片清晰
- 确保文字对比度足够
- 尝试使用更高分辨率的图片

### 文件打开失败
- 确保文件仍然存在于原位置
- 确保系统有对应的默认程序

## 技术细节

### 文件处理流程
1. 用户拖拽文件 → `ChatWidget.dropEvent`
2. 解析文件 → `FileParser.parse_file`
3. 创建附件对象 → `Attachment`
4. 显示文件卡片 → `FileCardWidget`
5. 发送消息时合并内容 → `AIClient.merge_attachments_to_content`
6. AI 接收完整上下文 → 生成回复

### 数据存储
- 附件信息存储在 `Message.attachments` 列表中
- 包含文件路径、名称、类型、大小、解析内容
- 会话保存时一并序列化

### 安全性
- 仅读取文件内容，不修改原文件
- 文件路径验证，防止路径遍历
- 文件大小限制，防止内存溢出
- 类型白名单，防止恶意文件
